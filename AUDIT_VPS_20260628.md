### C4 — explorer/server.py wallet app (MEDIUM)
New code adds Flask-based wallet with SQLite + JWT.

Issues:
1. **JWT secret fallback**: `JWT_SECRET=os.environ.get("JWT_SECRET", secrets.token_hex(32))`
   → If env var not set, random secret on every restart → all sessions invalidated.
   → Fix: require env var, fail to start without it.

2. **No rate limiting on login**: `/api/login` endpoint (assumed from pattern) has no rate limiting.
   → Brute-force risk on password endpoints.

3. **SQLite for production wallet**: Not suitable for concurrent access.
   → OK for dev/test, needs PostgreSQL/MySQL for production.

4. **Password hashing**: Need to verify `hashlib` uses proper KDF (bcrypt/argon2, not plain SHA256).
   → Check the actual hash implementation.

These are **non-consensus** issues (explorer only, not node).
